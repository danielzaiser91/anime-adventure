# Chronicles of the Celestial Blade

A browser-based anime RPG with branching narrative, turn-based combat, puzzles, and minigames. Fully playable in English and German. Installable as a PWA.

**[▶ Play now](https://danielzaiser91.github.io/anime-adventure/)**

---

## Story

You are the last wielder of the Celestial Blade — an ancient weapon capable of sealing the Void that is consuming the world. Joined by Luna (scholar), Ryu (warrior), and Suki (healer), you journey across a dying land to stop the Void Lord Malachar before the last seal breaks.

Your choices shape bonds, stats, and one of three endings.

---

## Features

- Branching dialogue with player-named protagonist
- Turn-based combat with timing mechanics and character skills
- Puzzles: seal assembly, void code cipher, memory sequence, weight balance
- Minigames: branch dodge, rhythm ritual, target lock, squad tactics, stealth
- 3 acts, multiple endings depending on your choices and stats
- AI-generated artwork — 26 backgrounds, 34 character sprites (6 expressions each)
- Full English / German localisation, auto-detected from browser
- Auto-save + 3 manual save slots
- PWA — installable, works offline after first load
- Lore glossary with highlighted keywords in dialogue

## Asset Generation

Game artwork is generated via [Replicate](https://replicate.com) FLUX.1-dev and served from GitHub Releases as WebP (3 sizes per image). SVGs in `public/assets/` are used as fallback.

```bash
# Generate all images (requires REPLICATE_API_TOKEN)
REPLICATE_API_TOKEN=r8_... npx ts-node --project tsconfig.scripts.json scripts/generate-assets.ts

# Upload to GitHub CDN and update imageUrls.ts
npx ts-node --project tsconfig.scripts.json scripts/upload-assets.ts
```

Existing files are skipped automatically — re-run safely after partial failures.

## Deploy

Push to `main` → GitHub Actions builds and deploys to GitHub Pages automatically. Discord `#news` channel receives an update embed with the changelog from `.github/release-notes.json`.

## Project Structure

```
src/
├── components/
│   ├── combat/          # CombatScreen, turn-based battle logic
│   ├── dialogue/        # DialogueBox, ChoicePanel
│   ├── exploration/     # ExplorationScreen, hotspot interaction
│   ├── minigames/       # 5 minigame types + router
│   ├── puzzles/         # 4 puzzle types (seal assembly, void code, memory, weight)
│   ├── screens/         # TitleScreen, EndingScreen, LoadingScreen
│   ├── layout/          # CharacterSprite, StatsHUD
│   └── ui/              # PauseMenu, LanguageSelector
├── data/
│   ├── story/           # act1.ts, act2.ts, act3.ts — all scenes and dialogue
│   ├── characters.ts    # skills and theme colours per character
│   ├── glossary.ts      # lore keyword definitions
│   └── imageUrls.ts     # CDN WebP map (generated by upload-assets.ts)
├── engine/
│   ├── storyEngine.ts   # scene lookup, ending selection
│   ├── combatEngine.ts  # damage, timing, defence calculations
│   ├── puzzleEngine.ts  # void cipher, validation
│   └── minigameEngine.ts
├── hooks/
│   ├── useImageUrl.ts   # CDN WebP → SVG fallback resolver
│   └── useScenePreloader.ts
├── i18n/en/ + de/       # UI and story translation files
├── store/gameStore.ts   # Zustand store — save/load/state
└── types/game.types.ts  # shared TypeScript types
```
